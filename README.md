# 🔍 CBIR System — Waste Classification (TrashNet)

A Content-Based Image Retrieval system for smart waste sorting, built with **classical image processing** techniques — no deep learning. Upload or capture a photo of a piece of trash, and the system returns the most visually similar images from the TrashNet dataset using a combination of colour, texture, and shape features.

Developed for **BMDS2133 Image Processing**, aligned with **SDG 11 & 12** (Sustainable Cities, Responsible Consumption) and **Malaysia MADANI** values.

## What it does

This system lets you query an image database by visual similarity instead of keywords. You provide an image (uploaded, captured from a webcam, or picked from the dataset), and the system returns the most similar images ranked by a weighted combined distance over 5 feature types.

It also supports **Rocchio relevance feedback** — you can mark results as relevant or not relevant, and the system refines the query and re-ranks.

The target use-case is automated waste sorting: given a photo of trash at a recycling facility, find similar objects from the reference dataset to help classify what it is and how it should be recycled.

## Features at a glance

- **5 classical feature types** covering colour, texture, and shape:
  - HSV Histogram (2048-d)
  - Colour Moments (9-d)
  - GLCM Haralick features (12-d)
  - Local Binary Pattern (10-d)
  - Histogram of Oriented Gradients (8100-d)
- **Weighted combined distance** with per-component normalisation
- **Rocchio relevance feedback** for iterative query refinement
- **Three input modes** in the web app: file upload, webcam capture, or pick from dataset
- **Full evaluation suite**: Precision@K, Recall@K, mAP, MRR, confusion matrix
- **Leave-one-feature-out ablation** to verify every feature contributes
- **Interactive Streamlit UI** with feedback buttons and a dataset stats dashboard

```
├── cbir_trashnet.ipynb     # Main notebook — indexing, evaluation, experiments
├── app.py                  # Streamlit web application
```

The notebook handles **offline indexing and evaluation** (run once). The app handles **online retrieval** (run every time a user wants to query).

## Requirements

- Python 3.9 or higher
- Dependencies:
  - `opencv-python` (cv2) — image loading, HSV conversion, HOG, calcHist
  - `numpy` — vectorised array operations, feature vectors
  - `pandas` — evaluation tables and stats dashboard
  - `matplotlib` — charts in the notebook and the app's stats tab
  - `streamlit` — web UI
  - `kagglehub` — downloads the TrashNet dataset
  - `jupyter` or `jupyterlab` — to run the notebook

Install everything with:

```bash
pip install opencv-python numpy pandas matplotlib streamlit kagglehub jupyter
```

## Setup and installation

### 1. Clone or download the project files

Make sure `cbir_trashnet.ipynb` and `app.py` are in the same folder.

### 2. Build the feature index (first-time setup)

The Streamlit app requires a file called `cbir_index.pkl` that contains the pre-extracted features for every image in the dataset. You build this by running the notebook:

```bash
jupyter notebook cbir_trashnet.ipynb
```

Then run every cell from top to bottom. The notebook will:

1. Download the TrashNet dataset via `kagglehub` (~2,527 images)
2. Validate the dataset (class distribution, dimensions, file integrity)
3. Extract all 5 feature types from every image
4. Normalise the features and save `cbir_index.pkl` in the project folder
5. Run the full evaluation suite and print the results

**Expected build time**: about 3–5 minutes on a typical laptop. The resulting `cbir_index.pkl` is around 100 MB.

You only need to do this **once**. After that, the app can be launched any number of times without rebuilding.

### 3. Launch the web app

Once `cbir_index.pkl` exists in the project folder, run:

```bash
streamlit run app.py
```

Your browser will open at `http://localhost:8501` with the app running.

## How to use

### Search tab

1. **Choose a query image** — three options:
   - **Upload your own** — select an image file (JPG, PNG, BMP, TIFF)
   - **Take a photo** — capture directly from your webcam
   - **Pick from dataset** — choose a category and image from TrashNet
2. **Adjust top-K** — how many results to return (default 10, max 20)
3. **Click Search** — the system extracts features from your query and returns the top-K most similar images, ranked by combined distance
4. **Give feedback (optional)** — tap 👍 or 👎 on each result. Tap again to un-select.
5. **Click Refine with feedback** — Rocchio refines the query vector and re-ranks. Repeat as many rounds as you want.

### Dataset & Stats tab

Shows dataset composition (class distribution, totals), feature vector dimensions, evaluation metrics, and the leave-one-feature-out ablation results.
